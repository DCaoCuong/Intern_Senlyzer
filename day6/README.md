## API Reference: Built-in Components & Hooks

1. Khởi động API Reference &  Component: Bắt đầu phần "API Reference". Tra cứu chi tiết API của component , tập trung vào cách nó tối ưu hóa prefetching.

2. Component API: Tra cứu chi tiết API của component (đã được giới thiệu về mặt tối ưu hóa trong Ngày 3) để hiểu sâu hơn về các props và cấu hình.

3. Tìm hiểu chi tiết API của component để nắm vững và so sánh các chiến lược tải (loading strategies) khác nhau mà Next.js hỗ trợ.

4. Tổng quan Hooks App Router: tìm hiểu chi tiết API của các hooks Next.js dành riêng cho App Router.

5. useParams & useSearchParams: Nghiên cứu chi tiết API và cách sử dụng useParams và useSearchParams để truy cập các tham số định tuyến động và query strings trong Client Components.

6. useRouter & usePathname: Nghiên cứu chi tiết API của useRouter (lưu ý sự khác biệt với Pages Router) và usePathname. Hiểu cách sử dụng chúng để truy cập trạng thái định tuyến trong Client Components.
